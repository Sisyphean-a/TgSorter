# 设置页密度收紧与标签修复设计

## 目标

在不改变设置页保存模型和工作台操作逻辑的前提下，进一步压缩设置页纵向空间，修复“来源会话”标签显示被裁切的问题，并明确工作台里“下一条”和“略过此条”的行为边界。

## 背景

当前设置页已经拆成折叠大类，但仍有几个实际使用问题：

- 底部保存条仍然采用卡片式容器，纵向占用偏大。
- `无引用转发` 仍带有解释文案，增加了不必要高度。
- 分类区域行间距与按钮间距仍偏松。
- `来源会话` 的表单标签显示异常，只露出下半部分，说明输入装饰或标签布局存在裁切。

## 约束

- 桌面端与手机端统一采用更紧凑的底部操作区，不做平台分叉。
- 不引入新的隐藏规则或折中兜底行为。
- 不改变设置保存、放弃、分类新增删除等业务行为。
- 工作台按钮行为不做重定义，只修正文案理解上的歧义。

## 方案选择

采用“保持交互不变，只收紧布局”的方案。

不在这次顺手重构工作台按钮的原因：

- `下一条` 与 `略过此条` 业务语义不同，当前代码实现边界清晰。
- 这次用户主要要求是设置页密度和标签修复，不应扩大范围。

## 变更范围

### 底部操作条

- 去掉卡片壳、圆角边框和额外包裹层。
- 保留状态文字与两个按钮。
- 使用接近顶部栏的扁平底栏样式，减少上下内边距。

### 工作流区域

- `无引用转发` 保留开关，仅移除副文案。
- 检查表单项装饰密度，修复 `来源会话` 标签裁切。

### 分类区域

- 收紧“新增分类”按钮下方空白。
- 收紧分类行容器的上下 padding。
- 收紧分类项之间的垂直间隔。

## 行为说明

### 下一条

- 只做浏览移动。
- 调用 `showNextMessage()`。
- 不写入操作日志。
- 不改变当前消息的分类状态。

### 略过此条

- 视为一次明确操作。
- 调用 `skipCurrent()`。
- 会写入 `skipped` 日志。
- 会把当前消息从当前处理序列中移除并展示下一条可见消息。

## 测试策略

重点覆盖：

- 设置页底部条不再依赖卡片容器样式。
- `无引用转发` 的副文案消失。
- `来源会话` 标签正常可见。
- 分类区在窄屏下仍不溢出。
