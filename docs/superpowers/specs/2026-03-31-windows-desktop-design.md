# TgSorter Windows 桌面端设计（Flutter + TDLib）

- 日期：2026-03-31
- 适用范围：同一代码库同时支持 Android 与 Windows
- 设计状态：已与用户确认

## 1. 目标

在保持现有 Android 端流水线能力不变的前提下，实现 Windows 桌面端应用体验优化：

- 桌面端采用双栏工作台布局；
- 支持键盘优先操作（快捷键可配置）；
- 业务语义不变（分类、跳过、撤销、批处理、重试）；
- 错误继续显式暴露，不引入静默降级。

## 2. 方案选型

采用方案 A：自适应布局 + 平台适配服务。

- UI 通过窗口宽度切换布局：移动端保留现状，桌面端启用双栏；
- 控制器与服务层尽量复用；
- 平台差异集中在 UI 壳层与平台能力适配点；
- 避免拆两套业务逻辑代码。

## 3. 架构设计

### 3.1 分层原则

- 业务层：`PipelineController`、`SettingsController`、`TelegramService` 不改变核心职责。
- 表现层：新增桌面工作台组件，移动端页面保持现有结构。
- 配置层：扩展设置模型支持快捷键配置，并持久化。

### 3.2 布局判定

引入 `LayoutProfile` 概念（通过宽度判断）：

- `mobile`: 沿用当前 `PipelinePage` 结构。
- `desktop`: 使用新双栏工作台。

不在业务控制器中引入平台判断，平台差异限定在页面层。

## 4. 桌面端信息架构

### 4.1 左栏（约 65%）

- 顶部：状态条（在线状态、拉取方向、处理中）；
- 中部：消息预览主区域（复用并扩展 `MessageViewerCard`）；
- 底部：分类 A/B/C 主操作按钮。

### 4.2 右栏（约 35%）

- 快捷键映射卡片（显示当前映射与启用状态）；
- 次级操作区（跳过、撤销、批处理 A、重试下一条）；
- 重试队列摘要（数量和下一条信息）；
- 最近日志列表（默认展示最近 20 条，可滚动）。

### 4.3 快捷键默认映射

- `1/2/3`：分类 `a/b/c`
- `S`：跳过当前
- `Z`：撤销上一步
- `R`：重试下一条
- `Ctrl+B`：批处理（A）

行为约束：仅主页面焦点时生效；输入框焦点时不触发。

## 5. 配置模型与持久化

### 5.1 模型扩展

在 `AppSettings` 增加 `shortcutBindings` 字段（不可变数据）。

- 提供默认映射；
- 提供更新单个动作快捷键与重置默认映射的方法。

### 5.2 仓储扩展

`SettingsRepository` 增加快捷键读写。

- 存储在 `SharedPreferences`；
- 对非法值回退默认映射；
- 不产生伪成功路径，页面展示实际生效值。

### 5.3 控制器扩展

`SettingsController` 增加：

- `saveShortcutBinding(action, key)`
- `resetShortcutDefaults()`

并在保存时进行冲突与合法性校验。

## 6. 异常处理

遵循“错误显式暴露”：

- TDLib/网络/鉴权/FloodWait 维持现有提示策略；
- 快捷键冲突或非法输入：保存失败并提示原因；
- 快捷键触发在离线/处理中/无消息场景下，沿用现有控制器状态约束。

## 7. 测试设计

- 单元测试：
  - `AppSettings` 快捷键默认值与更新；
  - `SettingsRepository` 快捷键读写与回退；
  - 快捷键动作映射到控制器调用。
- Widget 测试：
  - 宽屏渲染双栏；窄屏保持移动布局；
  - 快捷键映射区展示正确。
- 命令验证：
  - `flutter test`
  - `flutter run -d windows --dart-define-from-file=.env.local.json`（启动验证）

## 8. 交付边界

- 同一套代码支持 Android 与 Windows；
- Windows 双栏工作台；
- 快捷键配置（设置页可编辑与重置）；
- 核心流水线业务语义保持不变。
