# Overall UI Redesign Design

## 背景

当前项目的主要界面已经具备功能可用性，但整体观感仍然停留在“默认暗色后台”的阶段：

- 全局主题只做了基础暗色配置，缺少稳定的品牌语言。
- 工作台和设置页虽然都能使用，但视觉层级、留白、状态表达和组件风格并不统一。
- 手机端和桌面端更多是响应式适配，而不是围绕各自操作场景设计的专用工作流。
- 错误提示、剩余数量、日志、按钮反馈等状态分散在不同位置，精致感和顺滑度不足。

这会让产品显得“能用但不像成品”，也会削弱分类流程的连续性。

## 目标

- 建立统一的品牌化视觉系统，提升整体优雅感和完成度。
- 在不牺牲分类效率的前提下，让桌面端和手机端分别匹配其使用场景。
- 将核心工作台重构为更明确的处理流界面。
- 将设置页重构为更清晰的配置工作区，并与工作台共享同一套设计语言。
- 统一关键组件、状态反馈与过渡动效，让界面更顺滑、更稳定。

## 非目标

- 本轮不改动分类业务逻辑、TDLib 连接逻辑或消息处理策略。
- 本轮不为了视觉效果引入复杂特效、模拟态或隐藏失败的 fallback。
- 本轮不重写全部页面信息架构，只聚焦主工作流与统一界面基础。

## 方案对比

### 方案 A：纯视觉翻新

优点：

- 见效快，改主题和样式即可。

缺点：

- 只能改善颜色和圆角，无法解决操作流和状态层级问题。
- 手机端和桌面端仍然共用同一套布局思路。

结论：

- 不采用。

### 方案 B：纯工具化控制台

优点：

- 实现风险低，分类效率高。

缺点：

- 很难建立品牌感。
- 仍然会显得生硬、普通，无法解决“不够优雅”的问题。

结论：

- 不采用。

### 方案 C：创意感外壳 + 高效率工作流

优点：

- 品牌感集中在主题、层级、节奏和关键组件形态上。
- 桌面端和手机端都能围绕高频操作重新设计。
- 能同时解决美观度、完成度和操作体验问题。

缺点：

- 需要先补齐主题基础设施，再重构页面布局和组件。

结论：

- 采用。

## 整体视觉系统

产品采用“深色基底 + 电性青绿点缀”的品牌方向，但品牌色只用于关键交互和状态，不做满屏铺色：

- 页面背景使用带蓝灰与墨绿色倾向的深色，而不是纯黑。
- 主色使用电性青绿，承载选中态、主要动作、焦点反馈和进度提示。
- 表面分为四层：页面背景、工作区容器、内容卡片、浮层面板。
- 层级主要依靠明度差、描边和局部高光区分，而不是重阴影。
- 圆角、边框、间距和状态色统一收口，形成“锐利但不生硬”的工具气质。

## 交互布局

### 桌面端

桌面端使用双栏工作台：

- 顶部改为轻量品牌工具栏，承载产品名、工作状态、剩余数量、连接状态和设置入口。
- 主体区域左侧是消息查看区，右侧是分类操作区。
- 左侧优先服务于阅读、判断和媒体预览。
- 右侧集中放置分类按钮、快捷键提示、跳过、撤销和重试等决策操作。
- 日志、错误和次级信息降级到可展开区或次级面板，不与主流程抢层级。

### 手机端

手机端使用单手快速流：

- 顶部只保留标题、剩余数量和设置入口。
- 消息卡片成为页面核心，保持完整可读但不过度抢占纵向空间。
- 分类动作固定在底部拇指热区，减少滚动和来回切换。
- 次要工具和调试信息进入底部抽屉、折叠区或二级页，不与主按钮同层竞争。
- 页面节奏围绕“看一眼消息 -> 点一下分类 -> 继续下一条”设计。

### 设置页

设置页重构为配置工作区，而不是长表单堆叠：

- 继续保留单页结构。
- 通过卡片和分组组织“工作流设置 / 分类映射 / 连接配置 / 工具与调试”。
- 底部保存区保留为固定操作托盘，并强化“存在未保存更改”的可见性。
- 高风险操作、失败提示和确认弹窗使用统一的浮层风格。

## 关键组件

本轮重点重构以下组件：

- `消息卡片`：强化内容层级、媒体预览、来源信息和空状态表达。
- `分类按钮组`：统一尺寸、反馈、选中态和优先级，不追求花哨但要更有辨识度。
- `状态胶囊 / 状态条`：统一承载剩余数量、连接状态、同步中、失败、已修改等信息。
- `工作区卡片`：桌面端与设置页共用基础表面容器，统一层级和边界。
- `品牌工具栏`：替代默认普通 AppBar。
- `浮层组件`：统一确认框、选择框、错误提示和日志抽屉的样式与进入退出动效。

## 动效规则

动效只承担三种职责：提示层级变化、强化操作反馈、让切换更顺。

- 页面和面板切换采用短时长淡入或轻位移。
- 按钮点击、卡片选中和状态变化提供即时反馈，避免“点了像没点”。
- 展开收起、列表更新和设置区切换采用轻量过渡，减少机械感。
- 不引入夸张缩放、长距离位移或装饰性粒子动画。

## 状态反馈模型

统一为三层：

1. `即时反馈`
   - 用于单次操作完成、按钮响应、保存成功。
2. `上下文反馈`
   - 用于剩余数量、连接状态、未保存更改、处理中等持续状态。
3. `阻断反馈`
   - 用于保存失败、放弃更改、连接异常等必须决策的问题。

所有错误保持显式暴露，不做静默修正。

## 实施范围

第一轮按三层推进：

1. `主题基础设施`
   - 新增统一主题 token、页面容器、品牌工具栏、状态组件和交互动效常量。
2. `工作台重构`
   - 桌面端改为双栏工作台。
   - 手机端改为单手快速流。
   - 统一顶部状态区、消息卡片、分类操作区和次级信息层级。
3. `设置页重构`
   - 与新主题系统对齐。
   - 重整区块结构、状态提示和底部操作托盘。

## 验证标准

- 手机端分类动作更接近单手连续操作。
- 桌面端明确体现“左侧看消息、右侧做决策”的工作台结构。
- 设置页比当前更清晰，未保存状态和操作优先级更明确。
- 整体界面明显更有品牌感，但不会影响分类效率。
- 页面切换、按钮反馈和状态变化更顺滑且表达一致。
