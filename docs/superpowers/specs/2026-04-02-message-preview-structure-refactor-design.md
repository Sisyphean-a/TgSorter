# Message Preview Structure Refactor Design

## 背景

第一轮 UI 重构已经把工作台、设置页和全局主题拉到了统一的界面语言上，但消息预览链路仍然存在明显的结构风险：

- [message_viewer_card.dart](/f:/Github/TgSorter/lib/app/widgets/message_viewer_card.dart) 体量过大，职责混合。
- 卡片壳层、内容路由、文本/链接、图片/视频、音频、占位与错误态都挤在一个文件里。
- 预览行为本身已经不简单，继续在单文件上追加修改会显著提高回归成本。
- 第一轮已经确认消息卡片是工作台核心组件，因此第二轮优先做结构治理最划算。

## 目标

- 将消息预览链路按预览领域拆分成清晰文件边界。
- 保持现有分类工作流和媒体准备时机不变。
- 在不改变交互模型的前提下，顺手统一少量明显别扭的细节。
- 为拆分后的结构补足更明确的保护测试。

## 非目标

- 本轮不改 `PipelineController` 的媒体刷新和下载流程。
- 本轮不引入新的媒体类型或全屏预览能力。
- 本轮不把消息预览重构成复杂的 block renderer 或插件系统。
- 本轮不改变桌面端/移动端工作流。

## 方案对比

### 方案 A：最小拆分

做法：

- 将现有大文件机械拆成几个文件，但大部分逻辑原样搬运。

优点：

- 风险最低。

缺点：

- 只是“文件变多”，并没有真正建立清晰职责。
- 后续继续维护时收益有限。

结论：

- 不采用。

### 方案 B：按预览领域拆分

做法：

- 以“卡片壳层 / 内容路由 / 文本链接 / 媒体 / 音频 / 辅助组件”为边界拆分。

优点：

- 职责清楚，适合后续继续迭代。
- 风险可控，不会超出第二轮结构治理范围。

缺点：

- 需要补测试来保护拆分后的行为一致性。

结论：

- 采用。

### 方案 C：按渲染引擎体系重建

做法：

- 抽象成 preview block、media block、action block 等完整渲染体系。

优点：

- 长期结构最干净。

缺点：

- 设计过重，超出当前任务边界。

结论：

- 不采用。

## 目标结构

建议把消息预览链路拆成以下文件组：

1. `message_viewer_card.dart`
   - 保留对外入口。
   - 负责卡片壳层、头部、滚动容器、处理中遮罩。
2. `message_preview_content.dart`
   - 负责按 `MessagePreviewKind` 分发具体内容组件。
3. `message_preview_text.dart`
   - 负责文本、富文本实体、链接外跳。
4. `message_preview_link.dart`
   - 负责链接卡片 UI。
5. `message_preview_media.dart`
   - 负责图片、多图、视频、多视频组。
6. `message_preview_audio.dart`
   - 负责音频轨道列表、下载触发和播放状态。
7. `message_preview_helpers.dart`
   - 放时长格式化、占位态、局部错误态、通用小组件。

## 职责边界

### 卡片壳层

- 只关心卡片承载方式。
- 不直接编排具体媒体逻辑。
- 管理统一头部和处理中的遮罩。

### 内容路由

- 接收 `PipelineMessage?` 与当前加载状态。
- 决定渲染文本、链接、媒体还是音频。
- 不包含复杂 UI 细节。

### 文本与链接

- 富文本实体渲染与点击外跳。
- 链接卡片展示与跳转。
- 文本主导型内容之间共享相近的表现规则。

### 媒体

- 图片、多图、视频、多视频组。
- 统一媒体容器、时长信息、占位提示。
- 视频保留现有“点击播放后请求下载/初始化”的行为。

### 音频

- 音频播放状态与轨道列表。
- 多音频组按轨道渲染。
- 保持现有下载触发和播放器初始化时机。

### 辅助组件

- 时长文本、错误态、占位态、局部标签等。
- 避免在媒体和音频实现里重复拷贝格式化逻辑。

## 允许顺手优化的细节

- 统一头部、内容间距和局部留白。
- 收敛重复或不一致的文案。
- 统一占位态、错误态和时长信息的样式表达。
- 调整部分命名和参数接口，让子组件更清楚。

## 不允许改变的行为

- 不改变分类工作流。
- 不改变媒体下载触发时机。
- 不改变 `PipelineController.prepareCurrentMedia()` 的调用方式。
- 不新增新的交互入口或新业务能力。

## 测试设计

需要继续保护这些行为：

- 文本、链接、图片、视频、音频预览能正确渲染。
- 本地视频未就绪时仍显示点击播放入口，而不是自动加载。
- 视频缩略图、下载触发、错误态和时长显示不回归。
- 多视频组仍逐项展示，时长文本不重复出错。
- 音频预览仍支持下载前点击、播放切换和轨道列表。
- 卡片头部和空状态仍稳定存在。

## 完成标准

- [message_viewer_card.dart](/f:/Github/TgSorter/lib/app/widgets/message_viewer_card.dart) 不再承担所有预览职责。
- 每个新文件职责单一，名称和用途清晰。
- 视觉和交互只做轻微优化，不引入行为漂移。
- 相关 widget test 与静态检查全部通过。
