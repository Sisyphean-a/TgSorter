# Telegram 风格设置彻底重构设计

## 背景

当前设置页虽然已经具备“首页 / 二级详情页”两层结构，但整体质感仍明显偏向 Flutter Material 默认表单：

- 二级页核心编辑仍由 `DropdownButtonFormField`、`SwitchListTile`、常驻 `TextField` 驱动。
- 视觉令牌只覆盖了顶栏与列表底色，无法约束更细的行高、分割线、摘要文本、局部输入区与桌面容器。
- 过期文档仍引用 `settings_page.dart`，容易误导后续 AI 在错误入口上继续做表层修补。

本次目标不是继续“调样式”，而是把设置模块改造成更接近 Telegram 官方设置体验的专用系统。

## 目标

1. 移动端设置页整体质感贴近 Telegram Android 官方设置。
2. 桌面端设置页布局参考 Telegram Desktop，更舒适、更克制，不再像把手机表单横向拉宽。
3. 彻底移除设置模块对 Flutter 默认厚表单控件的主依赖。
4. 保留当前两层信息架构与显式保存语义，不引入第三级路由。
5. 清理过期文档与错误入口引用，统一当前真实实现边界。

## 非目标

- 不调整设置持久化格式。
- 不修改转发、标签、下载、快捷键等业务含义。
- 不把所有设置改成立即生效。
- 不引入新的路由框架。

## 方案概述

### 1. 保留两层设置架构，重写设置专用设计系统

保留现有 `SettingsNavigationController` 和页面级草稿保存模型，但将设置页的视觉与交互完全从 Material 默认表单中抽离。

新增一套设置专用组件族：

- 列表导航行
- 值展示行
- 开关行
- 多行描述行
- 危险动作行
- 局部信息卡
- 底部选择弹层
- 行内轻量输入弹层

这些组件负责设置页的主要交互，通用 `TextField` 只允许出现在局部编辑弹层内部，不再常驻详情页主列表。

### 2. 用“点行编辑”替换“页面常驻表单”

所有选择器、数字输入、文本输入统一改成 Telegram 式列表编辑：

- 点击行进入选项弹层或对话框。
- 当前值以右侧摘要或副标题形式展示。
- 修改后更新页面草稿，并在顶栏显式提供保存。

这样可以去掉当前大面积输入框和下拉框，让主页面恢复为 Telegram 风格的列表式设置页。

### 3. 分类与标签管理改成列表驱动的局部编辑

分类与标签不能升级成第三级页面，但也不能继续在详情页里堆编辑控件。

处理方式：

- 分类管理：显示为可点击的目标列表项，每项支持“变更目标”“删除”局部操作，新增通过对话框完成。
- 默认标签组：显示标签列表与新增入口，新增/删除在同页局部完成，但不让输入框长期裸露。

### 4. 桌面端改成 Telegram Desktop 风格设置容器

桌面端继续使用当前主壳层导航，但设置内容区改成：

- 居中窄列
- 更稳定的最大宽度
- 明确的页内滚动
- 更细的节奏和边距

避免当前设置页在桌面端横向铺满导致的廉价表单感。

## 视觉规范

### 移动端

- 蓝色顶栏，标题和返回图标高对比。
- 页面背景浅灰，列表容器纯白。
- 分组标题为轻量蓝色小标题。
- 列表行高约 54-58dp。
- 左图标浅灰或轻品牌蓝，不使用重描边卡片。
- 分割线为细线，起点对齐内容区，不全宽硬切。

### 桌面端

- 顶栏保持设置页专用风格，但页面主体采用居中面板。
- 面板宽度稳定，避免超宽输入区。
- 内容密度略高于移动端，但不变成工作台风格。

### 交互风格

- 开关使用 Telegram 风格配色与布局。
- 选择项右侧显示当前摘要，不直接显示下拉框。
- 文本/数字编辑通过弹层中的轻量输入区完成。
- 危险动作单独成组，使用明确危险色。

## 组件拆分

### 新增 / 重写的核心组件

- `settings_surface.dart`
  - 统一移动端与桌面端设置主体容器。
- `settings_list_tiles.dart`
  - 所有 Telegram 风格列表项。
- `settings_dialogs.dart`
  - 选项选择、文本输入、数字输入、确认对话框。
- `settings_switch.dart`
  - 统一设置页开关表现。
- `settings_section_block.dart`
  - 分组标题与列表块壳层。

### 调整的现有页面

- `settings_home_page.dart`
  - 目录页改成更像 Telegram 设置首页。
- `settings_screen.dart`
  - 切换为新的页面容器与编辑入口，不再直接承载厚表单。
- `settings_sections.dart`
  - 各领域只负责“领域行模型 + 事件”，不再拼装原始表单控件。
- `settings_common_editors.dart`
  - 旧编辑器整体下线或重写为弹层驱动组件。
- `settings_page_parts.dart`
  - 分类、会话、辅助块全部改成列表项式结构。

### 主题层

- `app_tokens.dart`
  - 为设置页增加更细的 Telegram 风格令牌。
- `app_theme.dart`
  - 收紧默认输入框样式影响范围，避免设置页被全局 Material 表单外观污染。

## 测试策略

1. 先新增失败测试，锁定新交互：
   - 首页与详情页不出现 `DropdownButtonFormField`
   - 详情页不出现常驻大输入框，改为点击行打开编辑入口
   - 选择器显示摘要值而非下拉框
   - 桌面端设置内容宽度被限制在窄列容器内
2. 再逐步重写组件，保持现有保存与错误显式暴露行为不退化。
3. 同步更新主题测试，锁定新的设置专用令牌与桌面容器风格。

## 文档清理

- `docs/ARCHITECTURE.md` 删除对 `settings_page.dart` 与 `sticky_action_bar.dart` 的设置页描述。
- 所有仍引用旧设置页入口的文档统一改为 `settings_screen.dart` 和新的组件边界。

## 风险

### 风险 1：视觉像 Telegram，但交互仍是 Flutter 表单

通过测试明确禁止 `DropdownButtonFormField` 和常驻厚输入框继续作为设置页主控件。

### 风险 2：桌面端直接照搬手机宽度，导致可用性差

通过设置专用桌面容器约束宽度和留白，而不是简单缩小同一套布局。

### 风险 3：重构过程中业务保存语义回退

保留现有页面级草稿、保存、放弃更改和错误显式暴露测试，重构只替换 UI 承载方式。

## 验收标准

1. 首页与详情页主视图中不再出现 `DropdownButtonFormField`。
2. 详情页主列表不再依赖常驻 `TextField` 作为主要编辑方式。
3. 选择、输入、开关都具备明显 Telegram 风格质感。
4. 移动端整体接近 Telegram Android 设置页。
5. 桌面端设置区具备接近 Telegram Desktop 的居中窄列体验。
6. 页面级保存、放弃、错误暴露行为保持正确。
7. 过期文档入口引用被清理。
