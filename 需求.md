# 📄 项目设计文档：Telegram 消息分发流水线 (TgSorter)

## 1. 项目概述

本项目是一个基于 Flutter 开发的 Android 移动端应用程序，旨在通过 Telegram 官方底层库 (TDLib) 以 **Userbot（用户客户端）** 的身份登录。

核心功能是自动拉取用户的“收藏夹 (Saved Messages)”中的消息，通过极简的 UI 界面逐条展示，并提供三个固定的分类按钮。用户点击按钮后，应用将自动执行“转发至目标对话 -> 删除原消息 -> 拉取下一条”的流水线操作。

## 2. 技术栈与核心依赖

- **框架**: Flutter (Dart)
- **目标平台**: Android
- **核心电报协议库**: `tdlib_dart` (或同类封装了 C++ TDLib 的 Flutter 插件)
- **状态管理**: `GetX` (推荐，便于快速实现响应式 UI 和路由管理)
- **本地存储**: `shared_preferences` (用于存储 3 个目标对话的 Chat ID 配置)

## 3. 核心架构与工作流

### 3.1 认证工作流 (Auth Flow)

由于使用 TDLib，必须实现完整的标准用户登录流程：

1. **检查状态**: 监听 `UpdateAuthorizationState`。
2. **输入手机号**: 处于 `authorizationStateWaitPhoneNumber` 时，渲染手机号输入框并调用 `setAuthenticationPhoneNumber`。
3. **输入验证码**: 处于 `authorizationStateWaitCode` 时，渲染验证码输入框并调用 `checkAuthenticationCode`。
4. **完成**: 进入 `authorizationStateReady` 状态，跳转至主流水线界面。

### 3.2 核心业务流水线 (Pipeline Flow)

核心业务必须严格遵循单向数据流（单次仅处理 1 条消息）：

1. **Fetch (获取)**: 调用 `getChatHistory`，目标 `chat_id` 设为用户自身的 User ID (即 Saved Messages)，`limit` 设为 1，拉取最新或最旧的一条消息。
2. **Render (渲染)**: 解析消息内容 (MessageContent)，在 UI 的大卡片中渲染（需至少支持文本 `MessageText` 和图片 `MessagePhoto` 的基础展示）。
3. **Action (动作)**: 用户点击底部 3 个分类按钮之一，触发异步任务。
4. **Forward (转发)**: 调用 `forwardMessages`，将当前 `message_id` 转发至按钮对应的 Target Chat ID。
5. **Delete (清理)**: 转发成功后，调用 `deleteMessages` (参数 `revoke: true`) 从收藏夹彻底删除该消息。
6. **Next (循环)**: 状态变量清空，重新触发 **Fetch** 步骤。

## 4. UI / UX 设计规范

界面需采用极客/工业风的极简设计，摒弃多余动画，追求操作效率。

- **屏幕上部 (占屏 60%) - 消息查阅区**:
  - 一个带有阴影和圆角的卡片 (Card)。
  - 内部使用 `SingleChildScrollView`，展示当前消息的具体内容。
  - 如果当前收藏夹为空，显示巨大的绿色勾选图标和文字：“收藏夹已清空，干得漂亮！”。
- **屏幕下部 (占屏 40%) - 分发控制台**:
  - 使用 `Row` 或 `Wrap` 均匀分布 3 个大尺寸的 `ElevatedButton`。
  - 按钮分别命名为“分类 A”、“分类 B”、“分类 C”（支持在设置中修改名称和绑定的 Chat ID）。
  - 点击按钮时，卡片区域需有轻微的 Loading 遮罩或透明度变化，防止重复点击。

## 5. 关键 API 映射说明 (TDLib)

> **⚠️ AI 助手请严格注意：本项目绝不能使用 `http` 请求 Telegram Bot API，必须全部使用 TDLib 的本地方法！**

- 获取自身信息 (用于确定 Saved Messages 的 Chat ID): `GetMe()`
- 获取历史消息: `GetChatHistory(chat_id: my_id, from_message_id: 0, offset: 0, limit: 1, only_local: false)`
- 转发消息: `ForwardMessages(chat_id: target_chat_id, from_chat_id: my_id, message_ids: [current_message_id], ...)`
- 删除消息: `DeleteMessages(chat_id: my_id, message_ids: [current_message_id], revoke: true)`

## 6. 异常处理与边缘边界

1. **风控与限流 (FloodWait)**: 当频繁调用转发/删除操作时，TDLib 可能会返回 `420 Flood Wait` 错误。需在全局拦截此类异常，通过 Snackbar 提示用户“操作过快，需等待 X 秒”，并暂停 UI 交互。
2. **网络异常断开**: 监听 TDLib 的连接状态 `ConnectionState`，断网时禁用底部 3 个按钮。
3. **复杂消息类型兼容**: 如果拉取到的消息类型为不支持渲染的复杂类型（如投票、位置、特殊文档），UI 卡片内应显示兜底文本：“[暂不支持预览的消息类型，请直接分类]”，但不影响转发和删除逻辑的正常执行。
